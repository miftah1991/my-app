Routing

  • navigating between page and display appropriated component/page on broswer
  • angular focus on SPA
  • help to navigate across the application form one view to another view
  • router check routes array to find the path
  • anngualr router is official library of angular
  • angular router redirect url, resovle date, run script when page aciveate or deactivate and lazy loading

steps, How angular router works

  • step: 1 parse the url and make tree of it
    • / divide ,() secondary rout : named router outlet ; matrix parameter ? query string # fragement // multiple secondary route
  • redirect if there is any redirect so navigate it
    • local redirect the back slash is not fount (/) replace single url segment absoulate redirect, start with / replace the entire url
  • step 3:identify the router state find the router segment from tree
  • step 4: Guard check the authentication
  • step 5 resovle run the resovler
  • step 6 activate the angular component
  • step 7 manage,listen for URL Change

Angular Forms

  • template driven form
  • model dirven forms (reactive forms)

template dirven form

  • easy to use
  • sutibale for simple scenarios
  • similar to angular 1
  • two way data binding using ngModel syntax
  • minimal component code

  • the model which is created in the .ts file is responsible for handling all the user interaction/validation
  • need to import reactiveformsmodule in our app.module.ts file

  • handle complex scenario

  • more componnent code less html markup
  • add elements dynamically
  • easy for code testing

setValue and patch value

how to set value to a control from ts page ,difference is there in set value we have to set value for all controls but in patch we can set value to one or more in patch is not required to set value to all controls

valuechanges() and statuschange()

these are properties of formcontrol, form array and form Group classes, it return observable instance and we subscribe to get data

value changes emit an eventy every time when the value of control change either using ui or programmaticallay

  • if we subscribe valuechanges of a form control instance we get latest value of that control whenever there is any change
  • if we subscribe valuechanges of a form array instance we get latest value of that array controls whenever there is any change
  • if we subscribe valuechanges of a form group instance we get latest value of the form controls whenever there is a change in any control of them

status change when the validation status of the control
